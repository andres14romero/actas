# Manual de integración de editores y generación de PDF

## Estructura modular

Cada carpeta contiene un módulo independiente:
- **html2pdf-tinymce/**: TinyMCE en frontend + exportación a PDF en navegador con html2pdf.js.
- **tinymce-backend/**: TinyMCE en frontend + exportación a PDF en backend con Puppeteer.

## ¿Cómo funciona cada parte?

### TinyMCE
- Es un editor WYSIWYG que permite crear y editar contenido HTML enriquecido.
- Puedes integrarlo en cualquier proyecto web con solo incluir el script y un textarea.

### html2pdf.js
- Convierte HTML a PDF en el navegador.
- Toma el HTML generado por TinyMCE y lo convierte en un PDF descargable.
- Limitación: el PDF es una "foto" del HTML, no texto real seleccionable en todos los casos.

### Puppeteer (Node.js)
- Permite generar PDFs en el backend, renderizando el HTML como lo haría Chrome.
- El PDF es casi idéntico al HTML original, soporta CSS avanzado, imágenes, tablas, etc.
- Ideal para generación masiva de documentos, guardar PDFs en el servidor, o enviar por email.

## ¿Cómo unir todo en un solo proyecto?

1. Desarrolla y prueba cada módulo por separado.
2. Cuando tengas claro qué librería y flujo te conviene, puedes unir los módulos:
   - Usa TinyMCE como editor principal.
   - Según el caso, exporta a PDF en frontend (html2pdf.js) o en backend (Puppeteer).
   - Si necesitas documentos masivos, puedes automatizar el envío de HTML al backend para cada cliente y guardar los PDFs en una carpeta.
3. Puedes crear un sistema donde seleccionas una plantilla, cargas los datos del cliente desde la base de datos, reemplazas las variables y generas el PDF personalizado.

## Consejos

- Siempre prueba el resultado visual del PDF con varios tipos de contenido.
- Documenta cada módulo y sus limitaciones.
- Si necesitas máxima fidelidad y control, usa Puppeteer en backend.
- Si solo necesitas exportar rápido desde el navegador, usa html2pdf.js.
